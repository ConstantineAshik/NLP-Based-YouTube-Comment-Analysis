Why is a single Train/Test split risky?

The standard "Holdout Method" splits data into two parts: Training (to learn) and Testing (to evaluate).

  • The "Lottery" Effect: If your dataset is small, a lucky split might put all easy cases in the test set (high accuracy), while an unlucky split puts all weird outliers in the test set (low accuracy).
  • Resource Waste: You never learn from the data held out for testing.

How Cross-Validation fixes the problem

k-Fold Cross-Validation rotates the role of the "Test Set" so every data point gets tested exactly once.

  • No data wasted: Everyone is used for training (in other folds) and testing (in their fold).
  • Stable Estimate: We average the results, canceling out the "lucky" and "unlucky" splits.
  • Trade-off: It takes k times longer to run.
