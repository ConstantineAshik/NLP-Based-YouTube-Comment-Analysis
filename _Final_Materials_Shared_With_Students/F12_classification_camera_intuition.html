Every Classifier is a Camera

A unified mental model for Machine Learning: Extract, Store, Apply.

1. The Unifying Analogy

Whether you are using a simple Threshold or a complex Neural Network, the machine learning process is mechanically identical to taking a picture. The camera Extracts light patterns, Stores them on a sensor/film, and allows you to Apply that image (display it) later.

In ML, models differ only in their resolution and flexibility.

3. Model Gallery

See how standard models fit into this exact framework.

1D Threshold / Logistic Regression
Analogy: Single Pixel

1. Extract

Takes a weighted sum of inputs (dot product). "How much 'light' is hitting the sensor?"
z = wˇ¤x + b

2. Store

Stores a simple vector of Weights ($w$) and a Bias ($b$). These define the angle and position of the separating line.

3. Apply

Compares the sum to a threshold (or passes through Sigmoid).
If z > 0, Class A

Decision Tree
Analogy: Low-Res Mosaic

1. Extract

Checks specific features one at a time. "Is x greater than 5?" Partitions space into boxes.

2. Store

Stores the Tree Structure: a hierarchy of nodes, split values, and leaf labels.

3. Apply

Drops the data point down the tree, following branches (True/False) until it hits a leaf node.

Feedforward Neural Network
Analogy: High-Res Telescope

1. Extract

Passes data through layers of neurons. Each layer extracts progressively complex features (edges → shapes).

2. Store

Stores massive matrices of Weights ($W$). Encodes a complex, non-linear function landscape.

3. Apply

Forward Propagation. Matrix multiplications followed by non-linear activations (ReLU/Sigmoid).
