The Impact of Scaling on Distance Metrics

Pairwise Distance Matrices

What is Euclidean Distance?

Euclidean distance calculates the **"straight-line"** distance between two items, $p$ and $q$, in a multi-dimensional feature space. It's a fundamental measure for clustering and classification algorithms.

The Formula

$d(p, q) = \sqrt{ \sum_{i=1}^{n} (p_i - q_i)^2 }$

The Core Problem

The choice of **feature scaling** drastically changes the resulting distances and, therefore, the clustering results. The same formula applied to differently scaled data will produce wildly different outcomes.

The following pages demonstrate this effect by calculating the Euclidean distance matrix for the same set of items using three different scaling methods.

Here, all numerical values are scaled to a **[0, 1] range**. This gives all features a more **balanced contribution** to the final distance. Case 3: Z-Score Scaling

Analysis

Values are standardized to have a **Mean of 0** and **Standard Deviation of 1**. This is a very common and robust method, as it is less sensitive to extreme outliers than Min-Max scaling. The resulting distances are again more balanced, reflecting the relationships across all features, not just one.

Visual Comparison: How Scaling Changes Relationships

To see the difference, let's plot the distances from **one item ("Honda")** to all other items. The change is dramatic.

Chart Analysis

Notice the "Raw" data (red bar). The distance from Honda to **AMS OIL** is $54.973$, an extreme outlier that dwarfs all other relationships. This completely skews any analysis.

After scaling (Min-Max and Z-Score), that distance becomes $3.000$ and $3.873$, respectively. These values are now in line with other distances, allowing the algorithm to perceive the more subtle relationships between other features.

Key Takeaway

Feature scaling is **not optional** in distance-based algorithms (like K-Means, KNN, or Hierarchical Clustering). It is a critical preprocessing step that defines the results. Using un-scaled data can lead to models that are biased, inaccurate, and based on arbitrary units.
