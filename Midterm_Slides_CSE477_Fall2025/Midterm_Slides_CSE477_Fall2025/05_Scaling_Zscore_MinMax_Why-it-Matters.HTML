The Problem: Unscaled Features Skew Distance

When features have vastly different scales (e.g., `people_count` from 1-3 vs. `brightness` from 0-1), distance calculations become dominated by the feature with the largest range.

Strategy 1: Handling Bad & Missing Data

Real-world data is messy. Automated detectors can make mistakes, leading to incorrect or impossible values that must be handled before analysis.

Scenario: A Contradictory Value

A new clip's `people_count` is automatically detected as 0, but we know a person is present. This value is **incorrect**, not a true zero.

Rule #1: Don't Invent Data. An impossible or contradictory value should be marked as missing (e.g., NA, null).

What to do with Missing Values (NA)?

Drop

Ignore the feature for that specific distance calculation.

Impute

Replace NA with a sensible statistic, like the median value for that scene type.

Model

Use other features (e.g., scene) to predict the most likely value.

Why Cleaning Matters

Treating a bad value like people_count = 0 as real data would make a solo performance seem falsely similar to an empty room, completely ruining our analysis. Proper cleaning ensures data integrity.

Strategy 2: Min-Max Normalization

This technique rescales every feature to a common range, typically [0, 1], giving each feature an equal opportunity to contribute to the distance calculation.

Applying the Formula

people_count (min=1, max=3)

L001: (1-1)/(3-1) 0.0 R001: (3-1)/(3-1) 1.0

brightness (min=0.30, max=0.45)

L001: (0.45-0.30)/(0.45-0.30) 1.0 R001: (0.30-0.30)/(0.45-0.30) 0.0

motion_magnitude (min=0.25, max=0.60)

L001: (0.25-0.25)/(0.60-0.25) 0.0 R001: (0.60-0.25)/(0.60-0.25) 1.0

The Normalized Vectors

L001' = (0, 1, 0) R001' = (1, 0, 1)

The Result: A Meaningful Distance

Recalculating the Euclidean distance with the normalized vectors yields a balanced and more meaningful comparison.

Normalized Differences

  • Difference 1: (1 - 0)² = 1.0
  • Difference 2: (0 - 1)² = 1.0
  • Difference 3: (1 - 0)² = 1.0

New Total Distance ≈ 1.732

Alternative: Z-Score Standardization

When your dataset is larger or contains outliers, Z-score standardization is often a more robust choice than Min-Max normalization.

✅ Pros

Less sensitive to outliers. Works well with many machine learning models that assume normally distributed data.

❌ Cons

Requires storing the mean and standard deviation from the training data to transform any new data, preventing data leakage.

Key Habits & Mindset

There is no single "best" method. The right choice depends on your data and your goal. The process is iterative and requires careful documentation.

Example: A Full Data Repair Workflow

Imagine a new clip (L002) is auto-detected with a clear error: people_count = 0 despite a person being visible.

  1. Flag the Error

    The `people_count` of 0 contradicts `human_present = 1`. Mark the value as missing (NA).

  2. Impute the Value

    Based on other 'home' clips, the median `people_count` is 1. We impute this value and add a flag like `people_count_imputed = True`.

  3. Recompute Normalization

    With the corrected value, we must update the dataset's min/max values and re-apply normalization to all features.

Document Everything

Keep track of imputations and the scaling methods used for reproducibility.

Visualize Your Data

Plot histograms before and after scaling to spot outliers or other issues.
