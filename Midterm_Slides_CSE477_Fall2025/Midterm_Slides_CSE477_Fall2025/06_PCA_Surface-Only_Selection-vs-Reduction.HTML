Dimensionality Reduction & Feature Selection

Shrinking our feature space to keep the signal and lose the noise.

The 'Why': Less Is Often More

Our goal is to reduce the number of features (the dimensions) without losing the crucial information that differentiates our music video clips.

Simpler Models

Fewer features are easier to understand and interpret.

Faster Training

Less data significantly speeds up algorithm execution.

Less Overfitting

Reducing noise helps models generalize better to new data.

Two Main Strategies

Feature Selection: Picking the best existing features. | Dimensionality Reduction: Creating new, fewer features.

Method 1: Feature Subset Selection

This is the most straightforward approach: we simply drop features that are redundant or don't provide much unique information.

Redundancy Example

  • human_present is perfectly determined by people_count.
  • If people_count > 0, then human_present = 1.
  • Action: We can safely drop human_present.

Low-Signal Example

  • A feature like shelves_visible might correlate strongly with scene=home.
  • If we already have the `scene` feature, shelves_visible adds very little new information.
  • Action: It can likely be dropped.

A Simple Recipe for Feature Selection

Filter Low Variance

Remove features that are almost constant (e.g., same value for 99% of clips).

Check Correlation

If two numeric features are highly correlated (e.g., |correlation| > 0.9), keep only one.

Document Decisions

Always record which features were dropped and the rationale behind it.

Method 2: Principal Component Analysis (PCA)

While feature selection drops features, PCA transforms them. It creates new, artificial features called Principal Components.

The Core Idea

PCA rotates the data to find new axes (Principal Components) that capture the maximum amount of variance. PC1 captures the most variance, PC2 the next most (uncorrelated with PC1), and so on.

This is especially powerful when you have many numeric features that are correlated, such as various measures of brightness, exposure, and highlights. PCA can often compress them into a single, more informative component.

Let's consider two perfectly correlated brightness features to see PCA in action.

PCA Steps & Results

  • Center Data: Subtract the mean (2, 4) from each point.
    • L001' = (-1, -2)
    • L002' = (0, 0)
    • R001' = (1, 2)
  • Find PC1: The direction of maximum variance is along the vector [1, 2].
  • Project Data: Convert two features into one new PC1 score.
    • L001 score ≈ -2.23
    • L002 score ≈ 0.0
    • R001 score ≈ 2.23
Interpretation: We reduced two redundant features into one new feature (PC1 score) with zero information loss. This single score still perfectly separates the three clips.

A More Realistic PCA Case

Let's consider our three normalized numeric features: `people_count`, `brightness`, and `motion_magnitude`. While we won't do the complex math, we can build an intuition for what PCA would reveal.

What to Expect from PCA

  • PC1: "Scene Activity"

    Would likely be a combination of `people_count` and `motion_magnitude`. The "stage" clip has high values for both, the "home" clip has low. This component could capture "scene energy" or "activity level."

  • PC2: "Lighting Condition"

    Might align more with `brightness`, capturing lighting differences independent of the number of people or motion.

Plotting our clips on a scatter plot of PC1 vs. PC2 would likely show "home" and "stage" clips forming distinct clusters.

When to Use Which Method?

The choice between Feature Selection and PCA depends on your project's goals and specific data characteristics.

Use Feature Selection When...

  • You need high model interpretability (original features are easier to explain).
  • You have features that are obviously redundant or useless (e.g., constant values).

Use PCA When...

  • You have many moderately correlated numeric features.
  • You prioritize predictive performance over direct feature interpretability.
  • Your primary goal is data preparation for clustering or distance-based algorithms.

Combine them: You can use feature selection first to remove obvious noise, then apply PCA to the cleaned, remaining set for further reduction.

Important Limits & Guardrails for PCA

Unsupervised Nature

PCA looks only for variance in features; it doesn't consider your labels (e.g., "home" vs. "stage"). It might preserve variance irrelevant to your specific task.

Hard to Interpret

Principal Components are mathematical combinations of original features. Their real-world meaning can be abstract and difficult to explain intuitively.

Avoid Data Leakage

Crucially, the PCA transformation (means, rotations) MUST be computed on training data only. Then, apply that exact same saved transformation to validation and test data.

Key Takeaways & What's Next

  • We simplified our feature set by selecting people_count and dropping redundant features.
  • PCA can compress multiple correlated features into fewer, more efficient components.
  • The choice between these methods involves a trade-off between interpretability and predictive power.
