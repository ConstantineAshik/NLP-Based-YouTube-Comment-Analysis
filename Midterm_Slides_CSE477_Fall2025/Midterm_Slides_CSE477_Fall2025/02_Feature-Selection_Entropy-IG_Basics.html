3. Entropy as a Pre-Clustering Signal

We can use Marginal Entropy (H(X)) as a simple measure of "feature variance" or "information." It tells us how much a feature's values change across the entire 7-row dataset.

Marginal Entropy H(X) per feature, computed from 7 rows (base-2).

How to Use This Chart:

Keep / Prioritize

Features with moderate entropy (e.g., 0.5 - 1.5) have variance that is interpretable and useful for clustering.

Drop / Deprioritize

Features with zero entropy (visc_w, container) are constants and must be dropped.

Use with Caution

Features with very high entropy (e.g., country, base) have high cardinality (many unique values). With only n=7, they risk dominating the distance calculation, making every row unique. Only keep them if you *want* geography (or granular base type) to be the primary driver of clusters.

4. Entropy-Based Feature Selection (After Clustering)

After you run an initial clustering (e.g., K-Means, Gower) and get cluster labels C, you can use entropy to see which features *best explain* those clusters.

1. Run Clustering

Get cluster labels C.

H(X|C)

2. Compute H(X|C)

Find conditional entropy of X given C.

IG

3. Find Info Gain

IG(X;C) = H(X) - H(X|C)

4. Rank & Select

Keep features with high IG or low H(X|C).
